# 殭屍生存卡牌 - Scrum 開發待辦清單

版本：0.2  
日期：2026-06-23  
用途：整理目前已完成、部分完成、尚未完成的開發任務，並依照 Scrum / Backlog 的方式安排優先順序。

這份文件是「開發管理文件」，不是 GDD 的替代品。  
GDD 負責描述遊戲規則與設計意圖；本文件負責描述「接下來要做什麼、先做什麼、怎樣算完成」。

## 產品目標

開發一個可遊玩的垂直切片版本，玩家可以：

1. 透過正規卡組進行回合與輪次循環。
2. 使用卡牌獲得或消耗食物、資源、能量、戰鬥值，之後加入士氣。
3. 在每輪之間處理事件選擇。
4. 使用卡牌倉庫、預設卡與替換系統調整卡組。
5. 面對由殭屍卡產生的殭屍威脅。
6. 在食物、士氣、戰鬥、探索與卡組規劃之間做出生存取捨。

## 優先級說明

- P0：核心循環必須完成，否則遊戲不能穩定測試。
- P1：遊戲主要特色與策略深度所需。
- P2：重要內容、進度系統或中後期功能。
- P3：美術、手感、音效、平衡工具與製作品質。

## 狀態說明

- 已完成：已實作，並且目前可用。
- 部分完成：已有原型，但規則、UI 或整合還不完整。
- 待辦：尚未實作。
- 阻塞：需要先決設計或其他系統完成。

## 完成定義

一個任務要被視為完成，必須符合：

- Unity Play Mode 可以正常運作，沒有 Console 編譯錯誤。
- 必要的 Inspector 欄位與 UI 綁定方式清楚。
- 功能走目前的正式資料流，不依賴一次性的測試捷徑。
- 不破壞既有的打牌、事件、替換、查看牌堆流程。
- 有清楚的手動測試方式。

### PB-P0-004 - 食物經濟 V1

狀態：V1 已實作，等待場景綁定與 Play Mode 驗證  
優先級：P0  
分類：生存經濟  
目標：讓食物成為第一個真正有壓力的生存資源。

規則：
- 食物可由卡牌與事件獲得
- 食物在輪末消耗，不是每回合消耗
- Demo 階段先使用單一固定消耗值
- 食物不足時，先觸發士氣下降
- 食物上限在輪末消耗後才檢查
- 超過倉庫上限的食物會被移除
- UI 顯示：目前食物 - 每輪消耗 / 倉庫上限

驗收：
- 輪末食物會被扣除
- 食物不足時觸發士氣下降接口
- 食物超過上限時會在輪末消耗後被移除
- UI 可清楚顯示食物壓力
